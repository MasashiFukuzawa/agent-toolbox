# Verification loop

## 手順

1. リポジトリの指示と `.agents/done.yml` から最小の quick 検証セットを特定する。
2. 意味のある変更単位ごとに、最も局所的で速い検証を実行する。
3. 失敗したら新しい変更を重ねず、最初の根本原因を直して同じ検証を再実行する。
4. 影響範囲が広がった時だけ関連テスト、型チェック、lint へ段階的に広げる。
5. 実行コマンドと結果を記録し、未検証事項を明示する。

検証を通すために assertion、型安全性、lint rule を無効化しない。完了報告の直前はこのスキルではなく `done` の最終ゲートを実行する。
