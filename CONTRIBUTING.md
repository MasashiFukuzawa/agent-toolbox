# Contributing

Thank you for improving Agent Toolbox.

1. Keep changes generic and free of private or organization-specific context.
2. Follow the conventions in [AGENTS.md](AGENTS.md) and [docs/skill-conventions.md](docs/skill-conventions.md).
3. Add or update trigger metadata and evaluation cases when skill routing changes.
4. Run the full development command set from [README.md](README.md#development).
5. Explain behavior changes, compatibility impact, and security implications in the pull request.

Do not include real credentials or production data in bug reports, fixtures, examples, or screenshots.
